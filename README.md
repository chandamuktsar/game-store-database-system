# Game Store Database Management System

This project is a relational database designed for a hobby game store that sells gaming products and rents table space for in-store games. The system allows customers to: **Purchase gaming products**, **Rent items**, **Book gaming tables**.

Since the business offers multiple services, the database is designed to manage:
- Customers and purchase history
- Product categories and attributes
- Table reservations and features
- Pricing tiers and price adjustments
- Rentals, employees, suppliers, and transactions

The goal of this project was to design a structured and normalized database that supports all business operations and to write SQL queries that generate meaningful insights about store activities.

## Project Features
- Designed normalized database structure
- Created tables with primary and foreign keys
- Implemented relationships between entities
- Managed rentals, purchases, payments, and reservations
- Built SQL queries for data analysis

## Main Tables
- Customers
- Products
- Product Categories
- Rental Records
- Purchases
- Payments
- Reservations
- Game Tables
- Staff Members
